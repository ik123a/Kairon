# Contributing to kairon

Thank you for your interest in contributing to kairon! We welcome contributions from developers of all skill levels.

## How to Contribute

### 1. Reporting Bugs
- Search existing issues to verify the bug hasn't been reported.
- Create a new issue describing the bug, including steps to reproduce and environment details.

### 2. Suggesting Enhancements
- Open an issue explaining the proposed change, use cases, and benefit to users.

### 3. Pull Requests
- Fork the repository and create a feature branch.
- Ensure your changes follow the existing style patterns and are well-commented.
- Add unit tests for new functionality.
- Verify that the CI tests pass locally.
- Submit a Pull Request targeting the `main` or `master` branch.

## Development Setup
- Follow the setup guidelines in the `README.md`.
- Ensure tests pass before committing.
